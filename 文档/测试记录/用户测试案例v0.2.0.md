**模式下拉框**位于聊天窗顶部标题栏，三个选项：

| 模式 | 身份 | 适用场景 |
|------|------|---------|
| **通用助手** | 通用开发者助手 | 日常问题、计算、通用编程 |

> **提示**：部分测试需要在 DST 助手和通用助手之间切换，对比行为差异。切换方法：点击下拉框选择即可，**选择会自动记住**（刷新页面后恢复上次的选择）。

## 测试 1：DST 助手 — grep 搜索源码

**测试目标**：验证 DST 模式下 LLM 能正确调用 `grep` 搜索 DST 源码，并解读结果。

**模式**：**DST 助手**

### 步骤

1. 确认下拉框选择的是 **DST 助手**
2. 输入消息：

   ```
   帮我查一下 DST 源码中 SetDomestication 函数在哪里定义的
   ```

3. 点击 **发送**

### 期望观察

| 位置 | 看到的 |
|------|--------|
| 聊天窗 | AI 气泡说"我来搜索"之类的话，然后出现 **紫色工具调用气泡**（`grep`），文件路径 `components\domesticatable.lua:48`，代码上下文 |
| 聊天窗 | 最后 AI 气泡总结函数定义位置，并解释`Domesticatable:SetDomesticationTrigger` 的功能 |
| Debug 统计条 | **工具调用** ≥ 1 |
| Debug StateGraph | `agent → tools → agent` 流转路径高亮 |
| Debug 步骤时间线 | `agent`（推理）→ `tools`（grep 执行）→ `agent`（总结）三步 |

### 关键验证：不能出现的行为

- ❌ AI 反问"DST 是什么？"
- ❌ AI 调用 `get_current_time`（问源码搜索却查时间）
- ❌ AI 说"我无法访问源码"然后直接放弃

### 正确现象截图判断

紫色工具气泡中应包含：

```
━━━ components\domesticatable.lua:48 ━━━
▶  48 | function Domesticatable:SetDomesticationTrigger(fn)
```

→ **DST 模式下 grep 路由正确。**

## 测试 2：模式对比 — DST 助手 vs 通用助手

**测试目标**：验证同一问题在两种模式下的行为差异——DST 模式更了解 DST 领域知识，通用模式更泛化。

### 2a：DST 领域问题

**模式**：先用 **DST 助手**，再切换到 **通用助手** 问同样的问题。

在两个模式下分别输入（**注意**：每次换模式后刷新页面，确保不同 thread_id 避免记忆干扰）：

```
DST 源码中 domesticatable 组件有哪些方法？帮我搜一下
```

| 观察维度 | DST 助手（预期） | 通用助手（预期） |
|---------|-----------------|-----------------|
| 是否调用 grep | ✅ 直接调用 | ✅ 也会调用（工具描述通用化了） |
| 回答风格 | 以 DST 开发视角解读，提到驯化系统 | 以通用代码搜索视角解读 |
| 对"domesticatable"的理解 | 知道这是 DST 的驯化组件 | 可能不会特别关联到 DST 游戏机制 |

> **注意**：通用模式下工具描述也已通用化，两种模式**都会调 grep**。差异主要体现在**回答视角**上。如果通用模式没有调 grep，说明工具描述还需要强化——请反馈。

### 2b：通用问题

**模式**：先用 **通用助手**，再切换到 **DST 助手** 问同样的问题。

在两个模式下分别输入：

```
Python 的装饰器是什么？一句话解释
```

| 观察维度 | 通用助手（预期） | DST 助手（预期） |
|---------|-----------------|-----------------|
| 是否正常回答 | ✅ 直接解释 | ✅ 直接解释 |
| 是否强行关联 DST | — | ❌ 不应该关联（已验证：DST 身份不污染无关问答） |

→ **DST 助手不干扰通用问题。**

### 2c：时间问题对比

在两个模式下分别输入：

```
现在几点了？
```

| 观察维度 | 通用助手（预期） | DST 助手（预期） |
|---------|-----------------|-----------------|
| 是否调用 get_current_time | ✅ 调 | ✅ 调 |
| 是否误调 grep | ❌ 不应该 | ❌ 不应该 |

→ **两种模式都能正确区分"问时间"和"搜源码"。**

## 测试 4：多轮对话记忆 + 线程隔离

**测试目标**：验证同一个 `thread_id` 能记住对话，不同线程之间互不可见。

**模式**：**通用助手**（记忆功能和模式无关）

### 4a：同一线程记忆

1. 输入：`我叫小明，我喜欢吃西瓜`
2. 输入：`我叫什么名字？我喜欢吃什么？`
3. 观察 AI 回答是否包含"小明"和"西瓜"

→ 如果记住了，第 2 条 AI 回复里会出现这两个词。

### 4b：线程隔离验证

浏览器 UI 目前只有一个默认 `thread_id`，验证隔离需要 curl：

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"我叫什么名字？直接回答即可","thread_id":"other-thread","mode":"general"}'
```

期望返回类似：`我不知道` / `你还没告诉我` —— **不会泄露测试 4a 中的 "小明"**。

→ **多轮记忆 + 线程隔离正常。**

## 测试 5：对话持久化（重启不丢失）

**测试目标**：AsyncSqliteSaver 让对话历史写入磁盘，服务重启后还能继续。

**模式**：**通用助手**（功能和模式无关）

### 步骤

1. 在浏览器发一条消息：`记住我的编号是 20260704`
2. 确认 AI 回复了（比如 "好的，我记住了"）
3. **重启 uvicorn**（`Ctrl+C` → 重新 `uvicorn src.main:app --reload`）
4. 浏览器刷新页面
5. 输入：`我刚才让你记住的编号是多少？`

### 期望观察

AI 回复包含 `20260704`

### 如果失败

- 检查终端是否有 `AsyncSqliteSaver 初始化完成，对话历史将持久化到磁盘`
- 没有这条日志 → 检查 `.env` 中 `OPENAI_API_KEY` 是否存在（lifespan 中 import settings 需要 `.env`）
- 检查项目根目录 `checkpoints.db` 文件是否存在

→ **对话持久化正常。**

## 测试 7：grep 搜不到时自动重试（仅 DST 模式）

**测试目标**：验证 DST 模式下，Agent 搜不到时会换搜索词重试（system prompt 中有此规则）。

**模式**：**DST 助手**

### 步骤

输入：

```
帮我查一下 DST 源码中 SetDomesticationPercent 这个函数
```

（这个函数名可能不存在——实际函数名是 `SetDomesticationTrigger` 或类似）

### 期望观察

- 第一次 grep 搜 `SetDomesticationPercent` → 未找到
- AI **不应直接放弃**，而应换搜索词重试（如搜 `Domestication` 或 `domesticatable`）
- 最终在重试中找到相关结果

→ **DST 助手具备搜索重试能力。**

> 如果 AI 第一次失败就放弃了，说明 system prompt 中的"grep 搜不到时换搜索词重试"规则没有被遵循——可能是模型差异导致，记录下来作为后续优化点。

## 故障排查快查表

| 现象 | 检查项 |
|------|--------|
| 聊天窗无响应 | 终端日志有无 `AsyncSqliteSaver 初始化完成`？没有则检查 `.env` |
| grep 工具说"未配置目录" | `.env` 中 `DST_SOURCE_DIR` 路径是否正确 |
| grep 工具找不出结果 | 搜索词确实在源码里吗（`PatternNotFound` 测试已通过 → 应该是真的搜不到） |
| DST 模式下调了 `get_current_time` | 模型差异——DeepSeek 可能忽略 system prompt，关注工具描述是否够明确 |
| 通用模式下调了 `get_current_time` | 同上 |
| 模式切换后样式没变 | 刷新页面确认 localStorage 是否被清除 |
| SSE 断连没有 cancel 日志 | 确认断连时机——消息刚发出去就关，别等回复到了再关 |
| 刷新后模式恢复成默认 | 浏览器是否禁用了 localStorage？检查 DevTools → Application → Local Storage |
