## 测试 1：grep → read_file 联动

**测试目标**：验证"grep 找位置 → read_file 读内容"的标准工作流。

**模式**：**DST 助手**

### 步骤

1. 确认下拉框选择的是 **DST 助手**
2. 输入：

   ```
   帮我找到 DST 源码中 SetDomestication 函数的完整定义，把它的代码贴出来
   ```

### 期望观察

| 位置 | 看到的 |
|------|--------|
| 聊天窗 | AI 先调 `grep(pattern="SetDomestication")`，紫色工具气泡显示匹配位置 |
| 聊天窗 | 接着调 `read_file(file_path="components/domesticatable.lua", start_line=..., end_line=...)`，紫色气泡显示带行号的完整函数代码 |
| 聊天窗 | 最后 AI 总结函数签名和逻辑 |
| Debug 统计条 | **工具调用** ≥ 2（grep + read_file），**循环** ≥ 3 |
| Debug StateGraph | `agent → tools → agent → tools → agent` 两轮 |

### 关键验证

- ✅ AI 在 grep 拿到文件路径和行号后，**主动调用 read_file** 而不是只看 grep 的 2 行上下文
- ✅ read_file 的 `start_line` / `end_line` 参数合理（不会整文件 200+ 行全读）
- ❌ AI 只调 grep 不看完整代码
- ❌ AI 跳过 grep 直接调 read_file（不知道文件路径）

## 测试 2：list_files 浏览目录 + 定向 grep

**测试目标**：验证"list_files 了解结构 → grep 定向搜索"的工作流。

**模式**：**DST 助手**

### 步骤

输入：

```
先帮我看看 DST 源码的 components 目录下有哪些文件，然后在这些文件中搜一下有没有和 "beefalo" 相关的代码
```

### 期望观察

| 位置 | 看到的 |
|------|--------|
| 聊天窗 | AI 先调 `list_files(directory="components")`，返回文件列表（带 📁/📄 标记） |
| 聊天窗 | 然后调 `grep(pattern="beefalo")` 搜索 |
| 聊天窗 | AI 根据 grep 结果总结 |

### 关键验证

- ✅ `list_files` 返回的目录结构清晰（目录在前、文件在后、有文件大小）
- ❌ AI 调了 `grep` 而不是 `list_files`（说明 docstring 还不够互斥）

## 测试 3：web_search 联网搜索

**测试目标**：验证 DuckDuckGo 联网搜索能返回 DST 相关结果。

**模式**：**DST 助手**（或通用助手）

> ⚠️ 本测试需要网络连接。

### 步骤

输入：

```
帮我在网上搜索一下 Don't Starve Together beefalo domestication API 的用法
```

### 期望观察

| 位置 | 看到的 |
|------|--------|
| 聊天窗 | AI 调 `web_search(query="Don't Starve Together beefalo domestication API")`，紫色气泡中显示 🔍 搜索结果（标题 + URL + 摘要） |
| 聊天窗 | AI 解读搜索结果，并**提示需要在本地源码中用 grep 验证** |

### 如果失败

- 终端可能显示 `未安装 duckduckgo_search 库` → `pip install duckduckgo_search`
- 搜索可能超时（网络问题）→ 换用更简短的搜索词再试
- DuckDuckGo 对中文搜索质量一般 → 用英文搜索词重试

## 测试 4：工具分工清晰度（多工具场景不混乱）

**测试目标**：验证新增工具后，Agent 不会在同类工具间混乱选择。

**模式**：**DST 助手**（当前 5 个工具，最容易混乱）

### 4a：grep vs read_file 区分

输入：

```
打开 components/domesticatable.lua 看看前 20 行
```

| 期望 | 不应该 |
|------|--------|
| ✅ 直接用 `read_file(file_path="components/domesticatable.lua", start_line=1, end_line=20)` | ❌ 调 grep（你已经知道文件路径了） |
| | ❌ 调 list_files（你要看文件内容不是列目录） |

### 4b：list_files vs grep 区分

输入：

```
DST 源码根目录下有哪些文件和文件夹？
```

| 期望 | 不应该 |
|------|--------|
| ✅ 调 `list_files(directory="")` | ❌ 调 grep（你不需要搜文件内容） |

### 4c：web_search vs grep 区分

输入：

```
DST 的 prefablist 文件在哪？源码里搜一下
```

| 期望 | 不应该 |
|------|--------|
| ✅ 调 `grep(pattern="prefablist")`（搜本地源码） | ❌ 调 web_search（你搜的是代码文件路径，不是查文档） |

## 故障排查

| 现象 | 检查项 |
|------|--------|
| read_file 说"文件不存在" | 路径是否是相对路径？`.env` 中 `DST_SOURCE_DIR` 是否配置？ |
| web_search 不工作 | `duckduckgo_search` 是否安装？网络是否正常？ |
| V4-Pro 返回异常 | 终端日志中 `agent_node mode=dst tools=[...]` 行是否正常？ |
| 工具路由混乱（grep/read_file 选错） | 记录具体案例，作为 v0.4.0 docstring 优化的输入 |
| v0.2.0 功能退化 | 检查 `src/agent/graph.py` 的 system prompt 是否保留了旧规则 |
