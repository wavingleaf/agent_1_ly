## 测试 1：grep → read_file 联动（用户）

**输入**：`帮我找到 DST 源码中 SetDomestication 函数的完整定义，把它的代码贴出来`

**消息流**（共 8 条）：

```
human → grep("SetDomestication") → 命中 domesticatable.lua:48
      → read_file(file_path="components/domesticatable.lua") → 读完 264 行
      → grep("function Domesticatable:Set") → 二次精确定位
      → AI 总结
```

**结论**：✅ 通过。LLM 正确执行了 grep → read_file → 补充 grep 的工作流。函数名实际是 `SetDomesticationTrigger` 而非 `SetDomestication`，LLM 发现差异后主动补充搜索，表现良好。

## 测试 2：list_files + 定向 grep（用户）

**输入**：`先帮我看看 DST 源码的 components 目录下有哪些文件，然后在这些文件中搜一下有没有和 "beefalo" 相关的代码`

**消息流**（共 14 条）：

```
human → list_files("components") + grep("beefalo")   ← 并行调用
      → list_files: 810 个条目（正确）
      → grep: 在 3982 个 Lua 文件中搜索（未限定子目录！）
      → grep("beefalo") → 50 条（再次全量）
      → grep("beefalo") + read_file + read_file → 30 条 + 两个文件
      → grep("domesticat") + grep("BEEFALO_DOMESTICATION")
      → AI 总结
```

**发现问题**：

> ⚠️ grep 工具缺少子目录限定参数。用户明确说"在这些文件中搜"（指 components 目录），但 grep 每次都是"在 3982 个 Lua 文件中搜索"而非限定在 components/ 下。LLM 意识到后尝试用 `read_file` 读具体文件来弥补，但费了额外 4 轮调用（从第 4 轮到第 7 轮）。

**修复**：`grep` 新增 `subdir` 参数（`src/tools/grep_ly.py`）。

| 改前 | 改后 |
|------|------|
| `grep(pattern)` — 始终从源码根目录搜索 | `grep(pattern, subdir="components")` — 可限定子目录 |
| 测试 2 用了 6 轮 grep（全量扫描 3982 文件 × 5 次） | 修复后应该只用 1-2 轮 grep（限定 810 文件） |

**结论**：⚠️ 发现问题 → 已修复。修复后 LLM 可通过 `subdir` 参数缩小搜索范围，减少 token 浪费。

## 测试 3：web_search 联网搜索（用户）

**输入**：`帮我在网上搜索一下 Don't Starve Together beefalo domestication API 的用法`
**追问**：`再试试联网搜索`

**消息流**（共 24 条）：

```
第 1 轮：web_search + grep 并行
  → web_search: 失败（DuckDuckGoSearchException → Bing 连接超时）
  → grep("domesticatable"): 成功，30 条匹配
  → read_file("domesticatable.lua"): 成功
  → grep("BEEFALO_DOMESTICATION") + grep("TENDENCY.") → 成功/未找到
  → grep("TENDENCY =") + grep("SetTendency") → 成功
  → read_file("constants.lua") + read_file("beefalo.lua") → 成功
  → AI 给出完整 API 分析

第 2 轮（用户追问）：web_search × 4 次
  → 两次"未找到结果"（DuckDuckGo 返回空）
  → 一次连接超时
  → 一次"未找到"
  → AI 诚实告知搜索不可用，建议用本地源码结论
```

**LLM 表现评价**：🟢 优秀。体现在：
- 联网搜索失败时，**不等待、不重试同一次搜索**——第一次失败立刻用本地 grep 兜底
- 联网与本地搜索**并行调用**（第 1 轮同时发了 web_search 和 grep）
- 联网彻底不可用时，**诚实告知**用户而非伪造搜索结果
- 本地结论不被联网失败影响——最终给出了完整的 API 分析

**DuckDuckGo 问题**：当前后端在中国大陆网络环境下不可靠（Bing 重定向超时、返回空结果）。暂不更换后端（见 TODO P2-5），因为：
- LLM 对联网失败的兜底处理已经足够好
- web_search 的定位是"辅助"而非"核心"

**结论**：⚠️ 已知限制（DDG 网络不可用），LLM 兜底优秀，暂不处理。

**测试数据**：[agent-chat-测试0.3.0 问题3-2026-07-06.json](agent-chat-测试0.3.0 问题3-2026-07-06.json)

### 测试 3b：web_search 联网搜索重测（用户，2026-07-07）

**改动**：`web_search` 新增 `DDG_PROXY` 代理支持（`socks5h://127.0.0.1:7890`）+ 指数退避重试（1s→2s→4s，最多 3 次）+ timeout 10→30s。

**输入**：同测试 3a

**消息流**（共 29 条）：

```
第 1 轮：web_search × 2（并行）
  → web_search("Don't Starve Together beefalo domestication API") → 空（查询太长）
  → web_search("DST beefalo domestication SetDomestication mod API Lua") → 空
  → LLM 同时启动 grep 兜底

第 2 轮：grep + web_search（并行）
  → grep("domesticat") → 30 条（全量 3982 文件）
  → web_search("DST beefalo domestication modding Klei") → ✅ 5 条成功！
    （无超时、无 DuckDuckGoSearchException）

第 3-7 轮：grep × 4（subdir="components"） + read_file × 3 + list_files
  → LLM 先后读取 domesticatable.lua / constants.lua / tuning.lua / debugcommands.lua / rideable.lua
  → 最终给出完整 API 分析（含核心文件、TENDENCY 定义、TUNING 常量、rideable 方法）
```

**关键发现**：

| 维度 | 改前 | 改后 |
|------|------|------|
| 连接超时 | ✅ 2 次 `ConnectError` | ✅ **0 次** — DDG_PROXY 生效 |
| 搜索空结果 | 2 次 | 3 次（含 2 次长查询 + 1 次无效词） |
| 成功返回结果 | 0 次 | ✅ **1 次 5 条** |
| grep subdir 使用 | 0 次 | ✅ **4 次限定到 810 文件**（components/） |
| 对话轮数 | 12 轮工具调用 | 14 轮（LLM 倾向多搜几轮确保完整） |

**DDG_PROXY 生效的明确证据**：消息 [6] 返回了 5 条结果且无异常，改前全部抛 `DuckDuckGoSearchException` 或 `ConnectError`。

**剩余的搜索质量问题**：DuckDuckGo 对长查询匹配能力有限——前两次用完整英文句子搜索返回空，第三次用短关键词 `modding Klei` 才成功。同时搜索结果的噪音较大（第一条结果 `DST: CTCP Đầu tư Sao Thăng Long` 是越南股票——股票代码也叫 DST）。这不是代码 bug，是搜索引擎本身的局限。

**LLM 表现**：与测试 3a 一致——联网与本地并行、失败不等待、成功搜到结果后仍用 grep 验证。

**结论**：✅ DDG_PROXY 代理链路打通，web_search 工具在中国大陆网络环境下可用。DuckDuckGo 长查询匹配差和噪音问题是搜索引擎本身的局限，暂不更换后端。对话轮数偏高是 DeepSeek V4-Pro 的行为特征，P2-4 `ModelCallLimitMiddleware` 可解决。

**测试数据**：[agent-chat-测试0.3.0 问题3 联网搜索优化后-2026-07-06.json](agent-chat-测试0.3.0 问题3 联网搜索优化后-2026-07-06.json)
