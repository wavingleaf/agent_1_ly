### 2. 模型切换：`deepseek-chat` → `deepseek-v4-pro`

**原因**：DeepSeek 官方 2026-07-24 硬切断 `deepseek-chat`（届时返回错误）。V4-Pro Agent 任务开源 #1。

改动：[.env](.env) `MODEL_NAME=deepseek-v4-pro`，前端 [debug_ui.html](debug_ui.html) 硬编码 badge 同步更新。

### 3. InjectedToolArg 改造

`grep`、`read_file`、`list_files` 的 `DST_SOURCE_DIR` 从隐式 `settings.` 依赖改为 `Annotated[str, InjectedToolArg]` 注入参数。LLM 看不到 `source_dir`，工具签名更显式、更好测试。

**踩坑**：裸 `InjectedToolArg` 类型标注会触发 Pydantic `Cannot generate a JsonSchema`。正确语法：`Annotated[str, InjectedToolArg]`。

改动文件：`grep_ly.py`、`read_file_ly.py`、`list_files_ly.py`

### 4. grep 子目录限定（`subdir` 参数）

**发现于测试 2**：用户说"在 components 目录下搜 beefalo"，但 grep 总是全量扫描 3982 个文件。新增 `subdir` 参数，LLM 可限定搜索范围（如 `subdir="components"` → 810 文件）。

改动：[src/tools/grep_ly.py](src/tools/grep_ly.py)。测试 3b 重测中 LLM 在 4/7 次 grep 调用中主动使用 subdir，效果良好。

### 5. web_search DDG_PROXY 代理 + 指数退避重试

**发现于测试 3**：DuckDuckGo 在中国大陆网络下连接超时（`DuckDuckGoSearchException → Bing ConnectError`）。

修复：新增 `DDG_PROXY` 配置（[src/config/settings.py](src/config/settings.py)），`web_search` 工具支持代理（`socks5h://127.0.0.1:7890`，Clash 默认端口）+ 指数退避重试（1s→2s→4s，最多 3 次）+ timeout 10→30s。

改后测试 3b 重测：无连接超时，成功返回搜索结果。剩余问题：DuckDuckGo 对长查询匹配差、搜索结果噪音大（搜索引擎本身的局限，非代码 bug）。

### 6. 对话导出/导入（单线程 JSON）

后端新增两个端点（[src/main.py](src/main.py)）：
- `GET /chat/export/{thread_id}` — 导出为 JSON（含消息列表 + 版本 + 时间戳）
- `POST /chat/import` — 导入 JSON → 还原 LangChain 消息对象（HumanMessage/AIMessage/ToolMessage）→ 写入 checkpointer → 返回新 thread_id

前端 [debug_ui.html](debug_ui.html) 标题栏新增 ⬇ 导出 / ⬆ 导入按钮，导出直接下载 `.json`，导入走文件选择器。

### 7. AI 气泡实时渲染修复

**问题**：Agent 回复后气泡不立刻显示内容，需刷新页面才出现。

**根因**：`processEvent` 中 `_placeholder = false` 在内层条件 `if` 之外，内容条件不满足时也被误置 false，导致 `onDone` 跳过补渲染。

**修复**：两处改动 [debug_ui.html](debug_ui.html)：
1. `_placeholder = false` 移入内层 `if`（仅确实渲染了内容才取消占位）
2. `onDone` 新增兜底：从 `steps` 数组倒序查找最后一条 `aiContent` 补渲染

## 踩坑记录

### InjectedToolArg 不能用作裸类型标注

`def grep(source_dir: InjectedToolArg = "")` → Pydantic `Cannot generate a JsonSchema for IsInstanceSchema`。正确：`def grep(source_dir: Annotated[str, InjectedToolArg] = "")`。

### `_placeholder = false` 置位时机错误导致气泡不渲染

`processEvent` 的 `updates` 分支中 `_placeholder = false` 在内层条件之外——内容条件不满足时也被误置 false，`onDone` 跳过补渲染。修复：移入内层 + onDone 从 steps 补渲染。

### DuckDuckGo 在中国大陆网络下连接超时

`duckduckgo_search` v8 默认 `backend='auto'` → 内部尝试多个后端 → Bing fallback 在中国大陆也被阻断。修复：`socks5h://127.0.0.1:7890` 代理 + 指数退避重试。连通性解决，但 DDG 对长查询匹配差和噪音问题仍存在（搜索引擎本身的局限）。

### `duckduckgo_search` 包版本

当前安装的是 v8.1.1。该包已重命名为 `ddgs`（v9+），旧版不再维护。当前版本在代理下工作正常，暂不迁移。
